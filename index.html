Key Implementation Details

1. Authentication Flow

User registers with email, password, and name. Password is hashed using bcrypt before storage. JWT tokens issued upon successful authentication and stored in Async Storage for persistence.

2. Real-time Communication

Socket.io establishes persistent WebSocket connections. Users join conversation rooms upon login. Messages are emitted to specific conversation rooms with real-time updates.

3. Database Models

Three main models: User (email, password, name, avatar), Conversation (type, participants, lastMessage), and Message (conversationId, senderId, content, attachment).
